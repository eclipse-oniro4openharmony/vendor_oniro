# Contributing to Eclipse Oniro for OpenHarmony

Thanks for your interest in this project.

## Project description

Eclipse Oniro for OpenHarmony is a distribution of
[OpenHarmony](https://www.openharmony.cn/) maintained at the Eclipse
Foundation. This repository contains the vendor/product-level configurations and build definitions of the distribution.

* https://projects.eclipse.org/projects/oniro.oniro4openharmony

## Eclipse Contributor Agreement

In order to be able to contribute to Eclipse Foundation projects you must
electronically sign the Eclipse Contributor Agreement (ECA):

* https://www.eclipse.org/legal/ECA.php

The ECA provides the Eclipse Foundation with a permanent record that you agree
that each of your contributions will comply with the commitments documented in
the Developer Certificate of Origin (DCO). Having an ECA on file associated
with the email address matching the "Author" field of your contribution's Git
commits fulfills the DCO's requirement that you sign-off on your contributions.

For more information, please see the Eclipse Committer Handbook:

* https://www.eclipse.org/projects/handbook/#resources-commit

## How to contribute

Report bugs and request features via this repository's GitHub issue tracker,
and submit changes as GitHub pull requests against the current release branch.
Every commit must be signed off (`git commit -s`) with an email address
covered by a signed ECA.
